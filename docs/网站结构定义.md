# MetaBlog 网站结构定义

本文档说明 MetaBlog 网站目录、源文件位置、配置文件、构建输出和主要页面结构。

## 1. 网站根目录

MetaBlog 的整站构建以一个网站根目录为输入。CLI 中通过 `-root` 指定：

```bash
metablog site build -root . -out out
```

初始化新网站时：

```bash
metablog site init -root my-blog -title "My Blog"
```

一个典型网站目录如下：

```text
my-blog/
  articles/
    my-first-article/
      main.tex
      section/
      fig/
      table/
      bib/
  asset/
    figs/
      circle_example.svg
  data/
    config.toml
    articles.toml
    about_page/
      main.tex
  web/
    static/
      fonts.css
      chroma-theme.css
      fonts/
  .metablog-cache/
  out/
```

主要目录：

| 路径 | 说明 |
| --- | --- |
| `articles/` | 文章源文件目录。推荐每篇文章一个子目录。 |
| `data/articles.toml` | 文章元数据列表，定义每篇文章的标题、分类、标签、源目录和主文件。 |
| `data/about_page/` | 关于页面源文件目录。 |
| `data/about_page/main.tex` | 关于页面主 LaTeX 文件。 |
| `data/custom_components/` | 自定义页面组件目录。 |
| `data/custom_components/page_footing.tex` | 全站页尾组件，构建时放入所有页面尾部。 |
| `data/custom_components/article_stat.tex` | 文章统计组件，构建时放入每篇文章作者信息下方。 |
| `data/config.toml` | 网站全局配置。 |
| `asset/` | 站点级资源目录，例如 logo 和 icon。 |
| `web/static/` | 站点静态资源目录，例如 `fonts.css`、`chroma-theme.css` 和字体文件。 |
| `.metablog-cache/` | 本地构建缓存目录，不应提交到仓库。 |
| `out/` | 默认构建输出目录，不应提交到仓库。 |

## 2. 网站配置

网站级配置位于：

```text
data/config.toml
```

示例：

```toml
title = "My Blog"
logo = "figs/circle_example.svg"
icon = "figs/circle_example.svg"
home_page_size = 10
article_list_page_size = 20
```

字段说明：

| 字段 | 说明 |
| --- | --- |
| `title` | 网站标题，用于顶栏和页面标题。 |
| `logo` | 站点 logo，路径相对于 `asset/`。构建时会复制到 `out/assets/site/`。 |
| `icon` | 浏览器 icon，路径相对于 `asset/`。构建时会复制到 `out/assets/site/` 并写入页面 `<link rel="icon">`。 |
| `home_page_size` | 首页每页文章数量；默认 10。 |
| `article_list_page_size` | 所有文章、分类文章列表和标签文章列表每页文章数量；默认 20。 |

## 3. 文章位置

文章源文件通常放在网站根目录的 `articles/` 下，每篇文章一个文件夹：

```text
articles/
  my-first-article/
    main.tex
    section/
      introduction.tex
    fig/
      main.pdf
    table/
      result.tex
    refs.bib
```

真正参与构建的文章由 `data/articles.toml` 声明。`articles/` 中存在文件夹并不代表一定会构建；只有配置文件中的未删除文章才会进入网站。

文章配置示例：

```toml
[[articles]]
title = "My First Article"
description = "文章简介。首页和文章列表会展示该字段，超过 1000 个字符会省略。"
author = "Author Name"
date = "2026-05-11"
category = ["Notes", "LaTeX"]
tags = ["LaTeX", "Blog"]
folder = "articles/my-first-article"
main_fig = "fig/main.pdf"
main_file = "main.tex"
deleted = false
```

字段说明：

| 字段 | 说明 |
| --- | --- |
| `title` | 文章标题。 |
| `description` | 文章简介，用于首页和文章列表。 |
| `author` | 文章作者。 |
| `date` | 文章日期，用于文章排序。 |
| `category` | 多级分类路径，例如 `["Paper", "L2O"]`。 |
| `tags` | 标签列表。 |
| `folder` | 文章源目录，相对于网站根目录。 |
| `main_file` | 文章主 LaTeX 文件，相对于 `folder`。 |
| `main_fig` | 首页和文章列表中展示的主图，相对于 `folder`。 |
| `deleted` | 软删除标记；为 `true` 时不参与构建和列表展示。 |

文章最终输出到：

```text
out/articles/<slug>/index.html
```

`slug` 优先来自文章配置中的 `slug` 字段；未显式设置时由标题生成。文章资源输出到：

```text
out/assets/articles/<slug>/
```

这样不同文章中的同名图片不会互相覆盖。

## 4. 关于（About）页面位置

关于（About）页面固定由下面的 LaTeX 文档生成：

```text
data/about_page/main.tex
```

关于页面源目录可以包含图片、表格、分章节文件等资源，例如：

```text
data/about_page/
  main.tex
  fig/
    avatar.pdf
  section/
    biography.tex
```

构建输出为：

```text
out/about/index.html
```

关于页面资源输出到：

```text
out/assets/about/
```

关于页面和文章页面使用同一套 LaTeX 解析与渲染流程，并会在整站构建时和文章构建并行启动。

## 5. 顶栏结构

整站页面共享统一顶栏。顶栏固定在页面顶部，不随窗口滚动。

顶栏左侧：

1. 站点 logo。
2. 网站标题。
3. 二者都链接到首页。

顶栏右侧：

1. 所有文章。
2. 标签。
3. 分类。
4. 关于。

顶栏的 logo 和 icon 来自 `data/config.toml` 中的 `logo` / `icon` 配置；网站标题来自 `title` 配置。

## 6. 首页

首页按日期从晚到早展示所有未删除文章。每页数量由 `home_page_size` 控制。

输出路径：

| 页面 | 输出 |
| --- | --- |
| 第 1 页 | `out/index.html` |
| 第 2 页及以后 | `out/page/<页码>/index.html` |

每篇文章在首页显示为一个文章块：

1. 顶部显示文章主图。
2. 主图下方显示文章标题，标题链接到文章页。
3. 标题下方显示发表日期和多级分类；每一级分类都链接到对应分类页面。
4. 下方显示 `description`；超过 1000 个字符时截断为 `...`。
5. 最下方显示标签行；标签图标紧贴标签列表左侧，每个标签链接到对应标签页面。

## 7. 所有文章页面

所有文章页面列出所有未删除文章，并按年份分组展示。文章列表页每页数量由 `article_list_page_size` 控制。

输出路径：

| 页面 | 输出 |
| --- | --- |
| 第 1 页 | `out/articles/index.html` |
| 第 2 页及以后 | `out/articles/page/<页码>/index.html` |

每个年份是一级分组。文章项左侧显示文章标题，右侧显示日期。

## 8. 标签页面

标签索引页面输出到：

```text
out/tags/index.html
```

它列出所有标签，并在标签后显示该标签对应的文章数量。

每个标签对应一个文章列表页面：

```text
out/tags/<tag>/index.html
out/tags/<tag>/page/<页码>/index.html
```

标签文章列表使用和“所有文章页面”一致的列表风格。

## 9. 分类页面

分类索引页面输出到：

```text
out/categories/index.html
```

分类来自每篇文章的 `category` 数组。数组顺序形成多级分类路径，例如：

```toml
category = ["Paper", "Learn to Optimize"]
```

会形成：

```text
Paper
└── Learn to Optimize
```

每个分类节点都会链接到该分类路径下的文章列表：

```text
out/categories/<category-path>/index.html
out/categories/<category-path>/page/<页码>/index.html
```

分类文章列表使用和“所有文章页面”一致的列表风格。

## 10. 文章页面

文章页面由对应文章目录中的主 LaTeX 文件生成。

输入位置由两个配置共同决定：

```text
文章主文件 = <folder>/<main_file>
```

例如：

```toml
folder = "articles/my-first-article"
main_file = "main.tex"
```

则输入文件为：

```text
articles/my-first-article/main.tex
```

文章页面会加入统一顶栏。正文部分由 LaTeX 转 HTML 渲染器生成。

## 11. 资源输出

站点级资源：

```text
asset/<rel>
```

构建后输出到：

```text
out/assets/site/<rel>
```

文章资源：

```text
articles/<slug>/<rel>
```

构建后输出到：

```text
out/assets/articles/<slug>/<rel>
```

关于页面资源：

```text
data/about_page/<rel>
```

构建后输出到：

```text
out/assets/about/<rel>
```

PDF 图片会优先转成 SVG。转换工具按顺序尝试：

1. `pdftocairo`
2. `mutool`
3. `inkscape`

普通图片和其他资源文件直接复制。

## 12. 增量构建

整站构建会对 about 页面和每篇文章执行文档级变更检测。

判断逻辑：

1. 递归扫描文档源目录中所有文件，得到最晚修改时间。
2. 读取该文档 HTML 输出文件的修改时间。
3. 如果源目录最晚修改时间不晚于 HTML 输出文件修改时间，则跳过该文档编译。

启用 `-force` 时会跳过该判断，强制重新编译 about 页面和所有文章。

资源文件有独立的逐文件 fresh 检测：

1. 如果输出资源文件已经存在、非空，且修改时间不早于源文件，则跳过复制或转换。
2. 否则重新复制，或将 PDF 重新转换为 SVG。

资源输出不参与文档级 freshness 判断；文档级判断只看 HTML 输出是否新于源目录。

## 13. 初始化命令生成的结构

`metablog site init` 可以在只有一个 `metablog` 二进制文件、没有任何源码仓库资源的情况下初始化网站目录。

它会创建：

```text
articles/
asset/figs/
data/about_page/
web/static/fonts/
```

并写入：

```text
data/config.toml
data/articles.toml
data/about_page/main.tex
asset/figs/circle_example.svg
web/static/fonts.css
web/static/chroma-theme.css
.gitignore
```

已有文件不会被覆盖。默认字体文件会从网络下载到 `web/static/fonts/`，其中包含正文字体和代码框使用的 Source Code Pro；如果使用 `-skip-fonts`，则只创建字体目录和 `fonts.css`。`chroma-theme.css` 是 `lstlisting` / `minted` 代码块的语法高亮主题样式。
